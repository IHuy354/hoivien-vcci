# 🚀 Setup Guide - CEO VCCI Frontend

## Prerequisites

Before you begin, ensure you have the following installed:
- **Node.js** 18+ 
- **pnpm** (recommended) or npm
- **Git**

## Quick Start

### 1. Install Dependencies

```bash
cd c:\DUAN\ceo-vcci-frontend
pnpm install
```

If you don't have pnpm installed:
```bash
npm install -g pnpm
```

### 2. Environment Configuration

Create a `.env` file in the root directory:

```bash
cp .env.example .env
```

Then edit `.env` and configure your environment variables:

```env
# API Configuration
NEXT_PUBLIC_API_URL=http://your-backend-url/api

# App Secrets (Generate random strings for security)
NEXT_PUBLIC_TMS_BUFFER=your_buffer_key_here
NEXT_PUBLIC_TMS_SECRET=your_secret_key_here
```

**Important**: Generate unique random strings for `NEXT_PUBLIC_TMS_BUFFER` and `NEXT_PUBLIC_TMS_SECRET`

### 3. Run Development Server

```bash
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Project Structure

This is a clean template with the following structure:

```
src/
├── app/
│   ├── (auth)/          # Auth pages - Add your login/register pages here
│   │   └── layout.tsx
│   ├── (main)/          # Main app pages - Add your app pages here
│   │   └── layout.tsx
│   └── layout.tsx       # Root layout
├── api/
│   ├── endpoints/       # API endpoint functions (account, upload)
│   └── models/          # Empty - Will be generated by Orval
├── components/          # Your React components
├── configs/             # App configuration (ACL, navigation, etc.)
└── ...
```

## Next Steps

### 1. Configure Your API

Update `orval.config.ts` with your OpenAPI specification URL:

```typescript
export default {
  api: {
    input: 'http://your-backend-url/api/swagger.json',
    // ... rest of config
  }
}
```

Then generate API client:
```bash
pnpm gen:api
```

### 2. Add Your First Page

Create a new page in `src/app/(main)/`:

```bash
# Example: Create a dashboard page
mkdir src/app/(main)/dashboard
# Create page.tsx in that folder
```

### 3. Update Navigation

Edit `src/configs/navigation.ts` to add your menu items:

```typescript
import { Home } from "lucide-react";

export const navigation: NavigationItem[] = [
  { 
    name: "Dashboard", 
    href: "/dashboard", 
    icon: Home, 
    subject: "InternNavigation" 
  },
  // Add more items...
];
```

### 4. Create Authentication Pages

Add login/register pages in `src/app/(auth)/`:

```bash
mkdir src/app/(auth)/login
# Create page.tsx for login page
```

## Folder Naming Conventions

- `(auth)/` - Route group for authentication pages
- `(main)/` - Route group for main application pages
- Groups in parentheses don't affect URL structure

---

## Tips

1. **Components**: Use shadcn/ui components from `src/components/ui/`
2. **State Management**: Use Zustand stores in `src/stores/`
3. **API Calls**: Use TanStack Query hooks with auto-generated endpoints
4. **Styling**: Use TailwindCSS utility classes
5. **Icons**: Import from `lucide-react`

## Troubleshooting

### Port 3000 already in use
```bash
# Kill process on port 3000 (PowerShell)
Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess | Stop-Process -Force
```

### Dependencies issues
```bash
# Clear cache and reinstall
rm -rf node_modules
rm pnpm-lock.yaml
pnpm install
```
